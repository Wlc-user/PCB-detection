🔌 CAN FD & 🌐 DoIP 详解

CAN vs CAN FD 核心差异

特性CANCAN FD
数据场8字节64字节
速率≤1Mbps≤8Mbps
CRC15位17/21位
帧开销47位67-133位
效率~50%~80%
// CAN FD 帧控制字段
FDF: 0=经典CAN, 1=CAN FD
BRS: 0=低速, 1=高速
ESI: 0=主动, 1=被动

// DLC 编码 (CAN FD)
0-8:  直接映射 (0-8)
9-15: 9,12,16,20,24,32,48,64

// 速率配置
仲裁段: 500kbps (传统CAN)
数据段: 2-8Mbps (CAN FD)

CAN FD 帧结构

SOF ID(11/29b) RTR/FDF BRS/ESI DLC DATA(0-64B) CRC ACK 
// CAN FD 周期报文 (CAPL)
variables
{
    message 0x200 fd_msg;
}

on start
{
    fd_msg.dlc = 64;
    fd_msg.fdf = 1;      // CAN FD格式

    // 填充64字节数据

DoIP 协议概述

DoIP (Diagnostics over IP) - 基于以太网TCP/UDP的诊断接入协议，ISO 13400标准

协议栈

// DoIP 协议分层
┌──────────────────────┐
│  Diagnostic Payload  │ UDS (0x10,0x27...)
├──────────────────────┤
│  DoIP Payload 0x8001 │ 诊断消息
├──────────────────────┤
│  DoIP Header (8B)    │ 协议头
│  ├─ Protocol Version │
│  ├─ Inverse Version  │
│  ├─ Payload Type    │
│  └─ Payload Length   │
├──────────────────────┤
│  TCP (13400) / UDP   │
├──────────────────────┤
│  IPv4 / IPv6         │
└──────────────────────┘

DoIP 报文示例

Version Inv.Version Payload Type Payload Length Payload Data... 
// 车辆识别请求/响应
0x0001: Vehicle ID Request    // UDP广播
0x0004: Vehicle ID Response   // VIN, Logical Address

// 路由激活
0x0005: Routing Activation Request
  - Activation Type: 01 (Default)
  - Logical Address Tester: 0x0E 0x00
0x0006: Routing Activation Response
  - Response Code: 0x00 (OK)

// 诊断消息
0x8001: Diagnostic Message
  - Source Address: 0x0E00 (Tester)
  - Target Address: 0x1000 (ECU)
  - UDS Data: 10 03 (会话控制)

CAPL 实现

// DoIP TCP 连接
variables
{
    socket doipSocket;
}

on start
{
    // 建立TCP连接
    doipSocket = tcpipConnect(
        "192.168.1.100",
        13400
    );
}

on tcpipTcpConnect(doipSocket)
{
    // 发送路由激活请求
    message * msg;
    msg.byte(0) = 0x02;  // Version
    msg.byte(1) = 0xFD;  // Inv Version
    msg.byte(2) = 0x00;  // Type MSB
    msg.byte(3) = 0x05;  // Type LSB
    tcpipSend(doipSocket, msg);
}

// 接收DoIP响应
on tcpipTcpReceive(doipSocket)
{
    byte data[100];
    int len = tcpipReceive(doipSocket, data);
    
    word payloadType = (data[2]<<8) | data[3];
    if (payloadType == 0x0006)
    {
        write("路由激活成功");
    }
}

总线协议对比

协议速率数据长度延迟成本典型应用
CAN125K-1M8B~1ms低车身控制
CAN FD1M-8M64B~0.2ms中动力/安全
FlexRay10M254B~0.1ms高X-by-Wire
LIN20K8B~10ms很低天窗/雨刷
100BASE-TX100M1500B~1μs高DoIP/娱乐
1000BASE-T1G1500B+~0.5μs很高视频/雷达

选型建议

// 选择依据
CAN:     成本敏感, 数据量小, 可靠性要求高
CAN FD:   大数据量, 高速传输, 需要OTA升级
FlexRay:  安全关键, 确定时延, X-by-Wire
Ethernet: 信息娱乐, ADAS, 网关, 远程诊断
