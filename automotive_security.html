🔒 车载网络安全

Automotive Cybersecurity · ISO/SAE 21434 · UNECE R155

💡 核心概念

保护车载网络免受攻击，包括 HSM安全芯片、SecOC真实通信、E2E端到端保护、安全启动 等关键技术。随着汽车智能化和网联化，网络安全已成为功能安全之外的另一大安全支柱。

📋 包含内容

🔐

HSM (Hardware Security Module)

硬件安全模块，提供密钥存储、加密运算、随机数生成等安全功能

  • 对称加密 (AES-128/256)
  • 非对称加密 (RSA/ECC)
  • 哈希运算 (SHA-256)
  • 真随机数生成器 (TRNG)
🔏

SecOC (Secure Onboard Communication)

安全车载通信，基于AUTOSAR标准，防止报文重放和篡改攻击

  • Freshness Value (FV) 新鲜值
  • Authenticator 认证码
  • 基于MAC的消息认证
  • 同步与验证机制
🛡️

E2E (End-to-End) 保护

端到端通信保护，确保数据从发送方到接收方的完整性和正确性

  • CRC校验
  • 序列号检查
  • Alive计数器
  • Profile 1-7 配置
🔒

安全启动 (Secure Boot)

确保ECU只执行经过认证的合法软件，防止恶意代码运行

  • 启动链验证
  • 数字签名验证
  • 信任根 (Root of Trust)
  • 回滚保护
✍️

OTA签名验证

软件升级包的数字签名验证，确保升级包来源可信且未被篡改

  • RSA/ECC 签名
  • 证书链验证
  • 哈希校验
  • 版本回滚检查
👁️

入侵检测 (IDS)

Intrusion Detection System，监控网络异常行为并告警

  • 报文频率异常检测
  • ID异常检测
  • 载荷异常分析
  • 安全事件记录

⚠️ 常见攻击类型 威胁

🔄 重放攻击 (Replay Attack)

攻击者截获合法报文后重新发送，欺骗ECU执行非预期操作

✂️ 中间人攻击 (MITM)

攻击者插入通信链路，窃听、篡改或阻断正常通信

🎭 伪装攻击 (Spoofing)

伪造ECU身份发送虚假报文，干扰车辆正常运行

💉 代码注入

通过漏洞向ECU注入恶意代码，获取控制权

SecOC 实现示例

/* SecOC 报文认证流程 */
// 1. 构建认证数据
uint8 dataToAuth[16];
memcpy(dataToAuth, pduData, pduLength);
memcpy(dataToAuth + pduLength, freshnessValue, 4);

// 2. 计算MAC (Message Authentication Code)
uint8 mac[4];
HMAC_SHA256(secretKey, dataToAuth, dataLen, mac);

// 3. 构建SecOC报文: [原始数据] + [FV] + [MAC]
SecOC_Transmit(pduId, securedPdu);

// 4. 接收端验证
if (SecOC_Verify(pduId, securedPdu) == E_OK) {
    // 新鲜值和MAC验证通过，处理报文
} else {
    // 验证失败，丢弃报文并记录安全事件
}
